# 🌦️ Weather Report Bot (บอทรายงานสภาพอากาศ)

เวิร์กโฟลว์ n8n ที่ซับซ้อนซึ่งส่งรายงานสภาพอากาศแบบบุคคลกับคำแนะนำไปยัง Discord ทุกเช้า

## ภาพรวม

**Weather Report Bot** เป็นเวิร์กโฟลว์อัตโนมัติที่ดึงข้อมูลสภาพอากาศแบบเรียลไทม์ วิเคราะห์สภาพอากาศตามฤดูกาลและสภาพแวดล้อม และส่งคำแนะนำที่ชาญฉลาดเกี่ยวกับการเตรียมตัวในแต่ละวัน คำเตือน และข้อควรระวังไปยัง Discord บอทนี้ใช้เครื่องมือที่ใช้กฎเพื่อให้คำแนะนำที่เหมาะสมกับบริบทเป็นภาษาไทย

## ✨ คุณสมบัติ

- **⏰ การทำงานอัตโนมัติรายวัน** - ตั้งเวลาให้ทำงานทุกเช้าเวลา 5:00 น.
- **🌡️ ข้อมูลสภาพอากาศแบบเรียลไทม์** - อินทิเกรตกับ OpenWeatherMap API
- **🔄 ป้องกันข้อมูลซ้ำ** - ตรวจสอบให้มีเพียงข้อความเดียวต่อเมืองต่อวัน
- **🧠 เครื่องมือที่ใช้กฎแบบอัจฉริยะ** - การจำแนกประเภทสภาพอากาศที่ขับเคลื่อนด้วย AI
- **🌍 การวิเคราะห์แบบหลายฤดูกาล** - จำแนกสภาพอากาศเป็น 5 ฤดูกาล:
  - ☀️ **ฤดูร้อน** (≥30°C)
  - 🌧️ **ฤดูฝน** (ฝน, ฝนปรอย, พายุ)
  - 🌱 **ฤดูใบไม้ผลิ** (20-29°C)
  - 🍂 **ฤดูใบไม้ร่วง** (10-19°C)
  - ❄️ **ฤดูหนาว** (<10°C)
- **📋 คำแนะนำตามบริบท** - ให้:
  - 🧳 คำแนะนำการเตรียมตัว
  - ⚠️ คำเตือนเกี่ยวกับสุขภาพและความปลอดภัย
  - 🚫 กิจกรรมที่ควรหลีกเลี่ยง
- **💬 การอินทิเกรต Discord** - ส่งข้อความที่จัดรูปแบบไปยังช่อง Discord
- **🌐 การรองรับหลายภาษา** - ค่าเริ่มต้นภาษาไทย (ภาษาไทย) สามารถขยายไปยังภาษาอังกฤษ

## 🔧 สถาปัตยกรรมเวิร์กโฟลว์

### ภาพรวมโหนด

|           โหนด            |                 วัตถุประสงค์                |       ประเภท     |
|---------------------------|------------------------------------------|------------------|
| **trigger every morning** |      ทริกเกอร์ตามกำหนดเวลา 5:00 น.         | Schedule Trigger |
|       **Set City**        |          ระบุเมืองเป้าหมาย (กรุงเทพ)          |     Set Node     |
|    **OpenWeatherMap1**    |         ดึงข้อมูลสภาพอากาศจาก API          |  OpenWeatherMap  |
|     **Normalize Data**    |    แยกและจัดโครงสร้างเขตข้อมูลสภาพอากาศ     |     Function     |
|  **Dedupe (1/day/city)**  |             ป้องกันข้อความซ้ำ               |     Function     |
|       **If Skip?**        | การกำหนดเส้นทางแบบมีเงื่อนไขตามการจัดเก็บข้อมูล |      If Node     |
|     **Rule Engine**       |                สร้างคำแนะนำ               |     Function     |
|    **Format Message**     |    จัดรูปแบบผลลัพธ์ด้วยอีโมจิและโครงสร้าง       |     Function     |
|     **Send a message**    |          โพสต์ไปยังช่อง Discord            |      Discord     |
|    **Noop (Skipped)**     |             จัดการข้อความที่ข้าม             |       NoOp       |

### การไหลของข้อมูล

```
trigger every morning
    ↓
Set City (Bangkok)
    ↓
OpenWeatherMap1 (fetch data)
    ↓
Normalize Data (extract fields)
    ↓
Dedupe (1/day/city) (check cache)
    ↓
If Skip? (decision)
    ├→ YES → Noop (Skipped)
    └→ NO ↓
      Rule Engine (analyze & recommend)
        ↓
      Format Message (format output)
        ↓
      Send a message (post to Discord)
```

## 📊 ตรรมชาติการวิเคราะห์สภาพอากาศ

### การจำแนกประเภทฤดูกาล

เวิร์กโฟลว์จัดประเภทสภาพอากาศปัจจุบันเป็นฤดูกาลโดยอัตโนมัติตามอุณหภูมิและประเภทสภาพอากาศ:

```
ถ้าสภาพอากาศมี [ฝน, ฝนปรอย, พายุ]
  → ฤดูฝน

ไม่เช่นนั้น ถ้า อุณหภูมิ ≥ 30°C
  → ฤดูร้อน

ไม่เช่นนั้น ถ้า อุณหภูมิ 20-29°C
  → ฤดูใบไม้ผลิ

ไม่เช่นนั้น ถ้า อุณหภูมิ 10-19°C
  → ฤดูใบไม้ร่วง

ไม่เช่นนั้น ถ้า อุณหภูมิ < 10°C
  → ฤดูหนาว
```

### ตรรมชาติของเครื่องมือที่ใช้กฎ

สำหรับแต่ละฤดูกาล เครื่องมือจะใช้กฎเฉพาะ:

#### 🌧️ ฤดูฝน
- **การเตรียมตัว**: ร่ม, เสื้อกันฝน, รองเท้ากันลื่น
- **คำเตือน**: ถนนลื่น, ทัศนวิสัยลดลง
- **ควรหลีกเลี่ยง**: พื้นที่ที่น้ำท่วม, การเดินทางที่ไม่ปลอดภัย

#### ☀️ ฤดูร้อน
- **การเตรียมตัว**: ดื่มน้ำให้เพียงพอ
- **คำเตือน**: ความเสี่ยงของความเหนื่อยล้า, ขาดน้ำ
- **พิเศษ**: ถ้า อุณหภูมิ ≥ 35°C → เตือนลมแดด, หลีกเลี่ยงแดดจัด

#### 🌱 ฤดูใบไม้ผลิ
- **การเตรียมตัว**: ชุดนุ่มสบาย, เสื้อผ้าแบบชั้น
- **คำเตือน**: การเปลี่ยนแปลงสภาพอากาศบ่อยครั้ง

#### 🍂 ฤดูใบไม้ร่วง
- **การเตรียมตัว**: เสื้อคลุม, แขนยาว
- **คำเตือน**: อากาศเย็น → ความเสี่ยงหวัดและไข้มากขึ้น

#### ❄️ ฤดูหนาว
- **การเตรียมตัว**: เสื้อผ้าหนา
- **คำเตือน**: ผลกระทบต่อสุขภาพ
- **พิเศษ**: ถ้า อุณหภูมิ ≤ 0°C → เตือนน้ำแข็ง, จำกัดเวลากลางแจ้ง

### ปัจจัยเพิ่มเติม

**การวิเคราะห์ความชื้น**:
- ≥ 90% → ชื้นมากเกินไป (ไม่สบาย)
- 30-90% → ปกติ
- ≤ 30% → ต่ำ (ความเสี่ยงผิวแห้ง)

**การวิเคราะห์ลม**:
- > 15 m/s → ลมแรงมากเกินไป (อันตราย)
- 10-15 m/s → ลมแรง
- < 10 m/s → ลมปกติ

## 🛠️ ตัวเลือกการปรับแต่ง

### เปลี่ยนเกณฑ์การจำแนกประเภทสภาพอากาศ
แก้ไขการตรวจสอบอุณหภูมิและสภาพอากาศในฟังก์ชัน **Rule Engine**

### ขยายการรองรับภาษา
เพิ่มการตรวจสอบภาษาและคู่การแปลในฟังก์ชัน **Rule Engine** และ **Format Message**

### เพิ่มกฎเพิ่มเติม
ขยาย Rule Engine ด้วยเงื่อนไขสำหรับ:
- คุณภาพอากาศ (PM2.5, PM10)
- ดัชนี UV
- ปริมาณเรณู
- ระยะมองเห็น
